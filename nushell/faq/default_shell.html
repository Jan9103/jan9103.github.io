Nushell as defult shell

As terminal default shell

This just works and is generally not a bad idea.

The only reason against it are:

  • I personally use 'bash' for quick things like opening files due to superior tab copmletion.
  • If nu would have some error for some reason you would have to fall back to something else.

As default $env.SHELL

This tends to break a lot of programs since they expect it to be a POXIS shell.

Some people do this and then have to apply fixes or workarounds to half the programs on their pc.

As login shell

I highly advice against this.

  • If nu breaks for any reason you need a recovory OS to get back in.
  • Some programs expect this to be POXIS and break if its not. (same as with $env.SHELL)

It is a better idea to add a 'if interactive try to open nu' to your login shell.

But i would not do that either due to my personal usecases for the login shell:

  • Open my WindowManager. (1 command for which the shell does not matter)
  • The WM froze due to to high load. (in this case i want it to start up as fast as possible with as little resources as possible -> dash)
