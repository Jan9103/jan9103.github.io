Numng usage

TL;DR

Applying and updating your shell config: 

numng -n b -u

Global arguments

These arguments have to be specified before the subcommand due to python limitation.

ArgumentShortTypeDescription
--nu-config
-n
flagLet the command target the shell-config instead of a project config

      
--verbose
-v
flagLet numng describe more precisely what it is currently doing

      
--package-file
-p
file-pathSpecify a numng.json file to use (defaults to 
numng.json
 in the current directory)

    


    
Build sub-command

    
sub-command: 
build
 (short: 
b
)

    
The process of applying/activating a configuration is called building.


    
Build arguments

    

      
ArgumentShortTypeDescription

      
--pull-updates
-u
flag (no extra arguments)Update the registry and packages if possible. Otherwise it will only contact the internet for new packages.

      
--overlay-file
-o
pathGenerate a file, which can be loaded using 
overlay use ./FILE.nu
, to enable the environment at the given path. (NOTE: a few things only work with script and not overlay files)

      
--script-file
-s
pathGenerate a file, which can be loaded using 
source ./FILE.nu
, to enable the environment at the given path. (in the shell-scope this defaults to 
SHELL_CONFIG_DIR/load_script.nu
)

      
--nupm-home
pathDefines a nupm_home directory. In the shell-scope this defaults to 
NUMNG_HOME/nu_config_nupm_home
.

      
--allow-build-commands
-b
boolean (
true
 or 
false
)		Override the 
"allow_build_commands"
 option in your base package

    


    
Init sub-command

    
sub-command: 
init
 (short: 
i
)

    
This command creates a new numng.json file to reduce the amount of writing you have to do.

    
When used in config-scope this command will also offer to install numng
